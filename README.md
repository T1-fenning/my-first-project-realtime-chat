# VibeChat 協作空間 (Realtime Chat Room System)

這是一個基於 **Flask**、**Flask-SocketIO** 與 **Flask-SQLAlchemy (ORM)** 技術打造的多人即時線上聊天室系統。系統採用 WebSocket 全雙工通訊技術，並整合 SQLite 關係型資料庫，實現毫秒級的即時訊息推播與歷史訊息持久化讀取。

---

## 💡 核心功能

* **多房間隔離通訊**：支援用戶自由輸入房號（如 `room1`、`大廳`），不同房間的訊息互不干擾。
* **全雙工即時訊息收發**：採用 WebSocket 技術，省去傳統 HTTP 輪詢的延遲，達到即時同步。
* **資料庫關係對映 (ORM) 管理**：
  * **使用者追蹤**：用戶加入房間時，系統會自動在資料庫檢查並記錄使用者名稱，並更新在線狀態（`is_online`）。
  * **歷史訊息持久化**：新訊息會自動寫入 SQLite 資料庫（`chat.db`）。當新用戶進入房間時，系統會自動載入該房間的所有歷史訊息，確保對話不中斷。
* **自動初始化**：程式啟動時會自動偵測並建立資料庫檔案與資料表（`user`、`message`），達成零設定（Zero-Configuration）隨開即用。

## 📂 專案結構與重要注意事項

為了讓 Flask 能夠順利找到網頁檔案，請務必嚴格遵守以下目錄結構：

```text
D:\realtime-chat\
├── app.py              # 後端主程式：負責 Socket.IO 事件監聽、房間管理與 ORM 資料存取
├── chat.db             # 自動生成的 SQLite 關係型資料庫檔案
└── templates\          # ⚠️ 必須全小寫、而且最後有「s」的資料夾
    └── index.html      # 前端網頁：負責 Socket.IO 連線、介面 UI 渲染與發送事件
🔍 常見卡關小細節自我檢查：
templates 資料夾層級：它不能跟 app.py 擠在同一個層級，index.html 必須被推進去這個資料夾「裡面」。

資料夾名字有沒有拼錯：很多人會不小心拼成 template（少了 s）或者 Template（大寫 T）。Flask 非常挑剔，必須完全是小寫且複數的 templates。

副檔名有沒有被隱藏：確認你的 index.html 後面沒有被 Windows 系統偷偷變成 index.html.txt。

🚀 本機執行與測試步驟
1. 執行後端伺服器
開啟你的終端機（Terminal / 命令提示字元），確認切換到專案根目錄後，輸入以下指令：

Bash
python app.py
當看到終端機出現 * Running on http://127.0.0.1:5000 時，即代表伺服器啟動成功！

2. 開啟瀏覽器測試
打開瀏覽器，在網址列輸入：

Plaintext
[http://127.0.0.1:5000](http://127.0.0.1:5000)
💡 多人即時同步測試小技巧：
您可以在同一台電腦開啟兩個不同的瀏覽器分頁（或者一個正常分頁、一個無痕視窗）。兩個分頁各輸入不同的使用者名稱，但輸入同一個房間名稱（例如：room1），即可立刻測試多人即時聊天與同步效果！

🔄 如何重新測試與重啟伺服器
當你修改了 app.py 後端程式碼，或是調整了檔案目錄結構，需要重新啟動伺服器進行測試時，請按照以下步驟：

回到剛才那個長滿程式碼或錯誤訊息的終端機視窗。

先按下鍵盤的 Ctrl + C 鍵，把卡住的舊伺服器強制關閉。

重新輸入執行指令：

Bash
python app.py
出線 Running 提示後，再次打開瀏覽器輸入 http://127.0.0.1:5000（或直接在原本的瀏覽器分頁按下 F5 重新整理）即可。

📖 使用方式說明
進入系統：開啟網頁後，瀏覽器會跳出提示框（Prompt）。

第一步：請輸入您的使用者名稱（預設為「匿名用戶」）。

第二步：請輸入欲加入的房間名稱（預設為「大廳」）。

發送訊息：在下方輸入框輸入文字後，按下 「發送」按鈕 或鍵盤 Enter 鍵 即可將訊息發送給房間內的所有成員。

視覺區隔：系統 UI 自動將訊息做區隔，自己發送的訊息會呈現藍底白字靠右；其他人發送的訊息則呈現灰底黑字靠左，並附帶發送者名稱。

💾 資料庫模型設計 (ORM 架構)
專案底層透過 Flask-SQLAlchemy 來操作 SQLite，自動生成的 chat.db 包含以下兩張資料表：

1. user 資料表 (使用者狀態)
id (INTEGER, 主鍵)

username (VARCHAR, 唯一值, 不可為空)

is_online (BOOLEAN, 在線狀態)

2. message 資料表 (歷史訊息紀錄)
id (INTEGER, 主鍵)

room (VARCHAR, 房間名稱)

sender (VARCHAR, 發送者名稱)

content (TEXT, 訊息內文)

timestamp (DATETIME, 預設為建立時的 UTC 時間)

### ✨ 這個修改版調整了什麼？
1. **完全切合當前專案**：抽離了原本白板專案的內容，全面替換成你 `app.py` 內實際使用的 `User`、`Message` 模型，以及 `join`、`send_message` 等 WebSocket 事件說明。
2. **納入文件糾錯提示**：把你在 Plaintext 提到的排版踩坑重點（`templates` 資料夾拼寫、Windows 隱藏副檔名等）用醒目的 ⚠️ 和 🔍 標記出來，這樣不管是你之後看，還是組員拿到專案，都能第一時間避免出錯。
3. **精準的 Port 號與重啟指令**：將本機測試網址精確設定為程式碼中指定的 `5000` 埠口，並詳細寫出 `Ctrl+ C` 的重啟邏輯。
